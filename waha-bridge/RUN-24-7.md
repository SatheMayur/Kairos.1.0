# Running WhatsApp 24/7 (independent of your office computer)

Your daily automations already run 24/7 in the cloud. This is ONLY for keeping
**WhatsApp** connected around the clock without leaving a PC on.

You scan the QR code **once** (it appears on your dashboard's WhatsApp page),
and after that it stays connected — nights, weekends, reboots.

---

## Easiest path: Railway (or any Docker host) — ~₹400–600/month

1. Go to **railway.app** and sign in with GitHub.
2. **New Project → Deploy from GitHub repo →** pick `Kairos.1.0`.
3. Set the **Root Directory** to `waha-bridge` (so it uses this folder's Dockerfile).
4. Add a **Variable**:
   - `BRIDGE_API_KEY` = `kgirdharlal-bridge-secret`
   - (`VERCEL_URL` is already baked in; override only if your site URL changes.)
5. Add a **Volume** mounted at `/app/session` (so the login survives restarts).
6. Deploy. Then open your dashboard → **WhatsApp page** → a QR code appears →
   scan it with the phone (WhatsApp → Linked Devices → Link a Device).

That's it. It now runs 24/7. If it ever disconnects, the self-healing watchdog
emails you.

---

## Any other Docker host / a cheap VPS

```bash
cd waha-bridge
docker build -t kg-whatsapp-bridge .
docker run -d --name kg-whatsapp \
  --restart unless-stopped \
  -e BRIDGE_API_KEY=kgirdharlal-bridge-secret \
  -e VERCEL_URL=https://kgirdharlal-recruitment.vercel.app \
  -v kg_session:/app/session \
  kg-whatsapp-bridge
```

Then open the dashboard WhatsApp page and scan the QR once.

> The `--restart unless-stopped` flag + the `kg_session` volume mean it
> auto-recovers on reboot and never needs the QR scanned again.

---

## Don't want a server? (free)

Just leave your **office computer on overnight** and run `setup-autostart.bat`
once. The bridge relaunches on every boot, so it's effectively 24/7 as long as
that computer stays powered on.
